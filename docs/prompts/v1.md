## 功能概述

提供一个 cli： `pi-feishu`，改命令有如下功能：

1. 加载 pi 运行时。基于 [pi sdk](https://pi.dev/docs/latest/sdk#createagentsessionruntime-and-agentsessionruntime)，这里还有一些 [sdk接入参考](https://github.com/earendil-works/pi/tree/main/packages/coding-agent/examples/sdk)
2. 包含一个TUI，可以通过 TUI 交互界面与 pi 运行时的交互
3. 连接飞书机器人，和飞书机器人通信。基于 [feishu channel sdk](https://github.com/larksuite/node-sdk/blob/feature/channel/docs/channel.zh.md#%E6%B5%81%E5%BC%8F%E5%9B%9E%E5%A4%8D)
4. 可以通过与飞书机器人对话实现 与 pi 运行时的交互
5. pi 运行时可由用户通过 --skill 参数加载自定义 skill；本项目不再内置 skills/ 目录 

## 功能详述

### 可以交互的内容

- session 相关管理 (list, new, delete)
- model 相关(model and think level) 管理 (list, select)
- 对话： extension command, prompt template, normal message

### TUI交互形式

- 参考 pi tui 的交互形式

### 飞书机器人交互形式

- session 相关管理
  - 用户发送 `/sessions` 消息
  - 以 card 消息形式返回 session 管理卡片，包含"当前session" "其他session列表" 和 "新建session"选项按钮。其中 其他session 列表中，每个session 旁有 "切换" 和 "删除" 选项按钮
  - 用户点击选项，pi 运行时执行相关选项内容

- model 相关管理
  - 用户发送 `/models` 消息
  - 以 card 消息形式返回 model 管理卡片，包含 当前model和think level， 其他 model 和 think level 列表。其他 列表中，每个item 旁有 "切换" 选项

- 对话
  - 以流式(打字效果)返回飞书机器人
  - 用户发送的消息，queue策略为 `{ streamingBehavior: "steer" }`
  - think, tool等在 TUI中能看到的，都要返回飞书机器人
